## route
- "/" : Login 페이지
- "/issues": open/closed SPA 페이지
- "/labels" : 레이블 페이지
- "/milestones" : 마일스톤 페이지
- "/news" : news 이슈 페이지

## 기능 요구사항
- [X] 로그인 확인 버튼을 누르면 /issues 페이지로 이동
  - 로그인 기능은 미구현 
- [X] route 설정
- [X] issues 페이지 스탕일링 
- [X] issues 데이터 요청 후 issues 카드 그리기
- [X] issue 해당 카드를 누르면 상세 페이지로 이동
